# Instagram Profile Mock 프로젝트 개요

이 프로젝트의 목적은 **실제 인스타그램 프로필 화면을 최대한 유사하게 재현한 개인용 웹 목업(Mockup) 사이트를 제작하는 것**이다. 단순히 정적인 디자인을 흉내 내는 것이 아니라, 실제 인스타그램을 사용하는 것과 비슷한 사용자 경험(UX)을 제공하는 것을 목표로 한다.

프로젝트는 HTML, CSS, JavaScript를 기반으로 구현하며, 별도의 백엔드 없이 동작하는 정적 웹사이트 형태를 지향한다. 완성된 결과물은 PC 브라우저뿐 아니라 스마트폰과 태블릿에서도 동일하게 사용할 수 있어야 하며, GitHub Pages와 같은 무료 정적 호스팅 서비스를 통해 배포하여 URL만으로 접속할 수 있도록 설계한다. 또한 PWA(Progressive Web App)를 적용하여 모바일에서는 홈 화면에 추가해 앱처럼 실행할 수 있도록 확장 가능성을 고려한다.

이 프로젝트는 **실제 인스타그램 계정을 대체하거나 연동하는 서비스가 아니라**, 사용자가 원하는 이미지와 정보를 이용해 자신만의 인스타그램 스타일 프로필을 표현하는 **개인용 목업**이다. 따라서 모든 데이터는 프로젝트 내부에서 관리하며 외부 API나 실제 인스타그램 데이터와는 연동하지 않는다.

## UI 및 디자인 목표

전체적인 레이아웃과 화면 구성은 실제 인스타그램 프로필 화면과 매우 유사한 느낌을 목표로 한다.

프로필 상단에는 프로필 사진, 사용자 이름, 인증 배지(선택), 게시물 수, 팔로워 수, 팔로잉 수, 소개글, 버튼 등이 배치된다.

그 아래에는 원형 하이라이트(스토리 하이라이트)가 가로 스크롤 형태로 표시된다.

하이라이트 아래에는 게시물(Grid), 릴스(Reels), 태그됨(Tag) 탭이 존재하며, 기본적으로 게시물 탭이 선택되어 있다.

게시물은 실제 인스타그램처럼 **3열(Grid) 구조**로 표시되며, 처음에는 화면에 보이는 일부 게시물만 표시되고 사용자가 아래로 스크롤하면 계속해서 다음 게시물을 볼 수 있어야 한다.

즉, 피드는 "3줄까지만"이 아니라 실제 인스타처럼 **끝없이 이어지는 세로 스크롤 구조**이다.

---

## 게시물 시스템

게시물은 각각 독립적인 데이터를 가진다.

게시물 하나는

* 이미지 1장
  또는

* 이미지 여러 장(캐러셀)

중 하나를 지원한다.

피드에서는 첫 번째 이미지만 썸네일로 표시되며, 이미지가 여러 장인 경우 우측 상단에 여러 장 아이콘이 표시된다.

사용자가 게시물을 클릭하면 게시물 상세 화면이 열린다.

상세 화면에서는

* 게시물 작성자
* 위치
* 여러 장 이미지 슬라이드
* 좋아요 수
* 설명
* 날짜

등이 표시된다.

여러 장 게시물은 좌우 스와이프 또는 화살표를 이용해 자유롭게 넘길 수 있어야 한다.

게시물을 닫으면 사용자가 보고 있던 피드의 **원래 스크롤 위치 그대로 복귀**해야 한다.

## 하이라이트 시스템

프로필 아래에는 하이라이트가 존재한다.

각 하이라이트는

* 제목
* 대표 이미지
* 여러 개의 스토리

로 구성된다.

하이라이트를 클릭하면 실제 인스타그램 스토리처럼 전체 화면 뷰어가 열린다.

스토리는

* 자동 재생
* 상단 진행 바
* 좌우 탭으로 이전/다음 이동
* 마지막 스토리 종료 시 자동 종료

기능을 지원한다.

스토리는 이미지뿐 아니라 향후 동영상도 지원할 수 있도록 확장성을 고려한다.

---

## 데이터 관리 방식

이 프로젝트는 별도의 서버를 사용하지 않는다.

모든 데이터는 프로젝트 내부의 JavaScript 데이터 객체(`data.js`)에서 관리한다.

프로필 정보, 게시물, 하이라이트 모두 하나의 데이터 객체에서 관리하며, JavaScript가 해당 데이터를 읽어 화면을 자동 생성한다.

새 게시물을 추가하거나 하이라이트를 추가할 때 HTML을 수정하지 않고 데이터만 수정하면 자동으로 화면이 갱신되는 구조를 목표로 한다.

## 개발 원칙

프로젝트는 유지보수가 쉽도록 구성한다.

HTML은 전체 구조만 담당하고,

CSS는 스타일만 담당하며,

JavaScript는 모든 동작을 담당한다.

코드는 기능별로 분리하여 작성하며, 추후

* 관리자 페이지
* 릴스
* 검색
* 댓글
* 좋아요 애니메이션
* 다크모드

등의 기능을 쉽게 추가할 수 있도록 확장 가능한 구조를 유지한다.

---

## 최종 목표

최종적으로 이 프로젝트는 단순히 "인스타처럼 생긴 페이지"가 아니라, **사용자가 자신의 사진과 데이터를 넣으면 실제 인스타그램 프로필을 사용하는 것처럼 느껴지는 개인용 웹 목업**을 만드는 것을 목표로 한다. 사용자는 프로필을 스크롤하여 게시물을 탐색하고, 게시물을 클릭하여 상세 내용을 보고, 여러 장의 이미지를 좌우로 넘기고, 하이라이트를 열어 스토리를 감상하는 등 실제 인스타그램과 매우 유사한 흐름으로 콘텐츠를 탐색할 수 있어야 한다.

---

### 개발 시 가장 중요한 원칙

이 프로젝트의 모든 구현은 **인스타그램의 사용 경험을 참고하되, 특정 상표나 보호받는 디자인 요소를 그대로 복제하는 것이 목적이 아니라, 개인용 목업으로서 자연스럽고 완성도 높은 UI와 UX를 제공하는 것**을 목표로 한다. 코드 역시 기능별로 체계적으로 구성하여 이후 새로운 기능을 추가하거나 수정하기 쉽도록 설계한다.
